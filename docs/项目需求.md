企业应用管理工具，该工具需部署并运行在Linux操作系统上。其核心目标是实现一个兼容多产品、多服务的统一平台，用于集中管理应用的发布、安装、部署、升级与运维。

**技术栈要求：**
- **后端：** 采用Spring Boot框架，构建RESTful API接口，确保接口的规范性、安全性与高性能。
- **前端：** 采用Vue 3框架，配合Element Plus或Ant Design Vue组件库构建现代化、响应式的用户界面。
- **数据库：** 使用内嵌的SQLite数据库，系统应能自动生成数据库文件及所需的数据表。

**服务端核心功能模块详细要求：**
1.  **应用管理：**
    - 创建并发布应用版本。
    - 一对多管理多个迭代版本。

2.  **应用版本包生成：**
    - 提供界面供用户勾选当前服务器上的多个特定目录或文件或者上传安装包。
    - 允许用户指定版本号、操作系统、前置依赖应用。
    - 提供可定制的systemd `service`文件模板，用于服务启动管理。
    - 根据以上配置，打包生成一个完整的、可部署的服务版本安装包。
    - 提供客户端监控方法（基于service status/http/nacos）

3.  **产品管理：**
    - 支持产品的创建与版本发布。
    - 产品需有独立的版本号管理。
    - 产品版本可绑定多个指定版本的应用。
    - 实现基于产品版本的一键安装能力。

4.  **服务广播：**
    - 支持同网段下广播服务端源地址。

5.  **非功能性需求：**
    - 工具本身应提供简易的安装与启动脚本，确保在Linux环境下的便捷部署。
    - 保证系统的高可用性与可扩展性，以支持未来更多产品或服务的接入。
    - 前端与后端需进行充分的错误处理与用户友好提示。
    - 不需要用户登录与鉴权


**客户端核心功能模块详细要求：**
1.  **应用商店：**
    - 自动扫描服务端源地址。
    - 从服务端拉取产品/应用、一键部署、升级、卸载。
    - 可查看安装进度、日志

2.  **应用监控：**
    - 基于服务端发布的监控方式提供服务运行状态监控

3.  **非功能性需求：**
    - 工具本身应提供简易的安装与启动脚本，确保在Linux环境下的便捷部署。
    - 前端与后端需进行充分的错误处理与用户友好提示。
    - 不需要用户登录与鉴权
